<Update label="June 7, 2026">

## New features

**Skills, tools, memory, auto mode, and connectors.** Babbily can now personalize replies with persistent [memory](/features/memory), use [tools and skills](/features/tools-and-skills) automatically when they help, and pull live context from [connected apps](/features/connectors). Auto mode picks the right capability for the task without manual toggling. See the [built-in tools reference](/features/built-in-tools-reference) for what each tool does.

**Visual replies in chat.** Replies can now include maps, charts, stat cards, weather, QR codes, code diffs, citations, images, and videos when they help answer the question. Learn more in [visual replies in chat](/features/visual-replies-in-chat).

**API usage billing.** A new monthly [API usage budget](/features/usage-budget) replaces fixed message limits. Track what you've used, see what counts, and know when it resets — all from the chat sidebar.

**Team admin controls.** Workspace owners and admins can manage invitations, roles, billing visibility, and the shared team budget from one place. See the [team admin guide](/features/team-admin-guide).

## Updates

**Clearer context indicator.** The [context window](/features/context-window) tracker now updates more reliably during long chats so you can tell when Babbily will compact older turns.

**Budget refresh.** Your remaining budget refreshes faster after each turn, so the number you see matches what you've actually used.

**Customer-friendly privacy controls.** Review what Babbily stores, how to clear memory, when to use temporary chats, and how to disconnect connectors in the [data controls and privacy FAQ](/features/data-controls-and-privacy-faq).

**Faster ways to get help.** Browse [task recipes](/guides/task-recipes) for common workflows, check [common issues](/troubleshooting/common-issues) for self-serve fixes, or [contact support](/troubleshooting/contact-support) with the right details ready.

## Fixes

- Resolved a stall when the token-limit warning appeared mid-send.
- Restored model fallback so [chat](/features/chat-and-models) keeps working when a preferred model is briefly unavailable.
- Fixed scrolling and layout glitches in [temporary chats](/features/temporary-chat) and on tablet viewports.
- Sign-in via Google and Apple is now more reliable on mobile.

</Update>
