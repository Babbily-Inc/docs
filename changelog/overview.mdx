<Update label="June 10, 2026">

## Updates

**Personal plans preserved when you join a team.** If you're added to a team while you have a personal subscription, your personal plan is kept intact and you keep access to manage it from the [billing portal](/features/billing-and-plans).

## Fixes

- Team checkout now validates the [seat minimum](/features/teams) against the selected price, preventing invalid orders at checkout.
- New users on the free tier see their [usage budget](/features/usage-budget) anchor correctly on their first chat, so the monthly reset date is accurate from day one.

</Update>

<Update label="June 9, 2026">

## Claude Fable added to Babbily

Claude Fable 5 is now available on Babbily — with a 1 million token context window.

Fable is a Mythos-class model built for long-running, complex, and asynchronous work, with robust safeguards designed for serious tasks. That means fewer interruptions, less context loss, and more room to think through big projects from start to finish.

Available now on Babbily.

</Update>

<Update label="June 7, 2026">

## New features

**Skills, tools, memory, auto mode, and connectors.** Babbily can now personalize replies with persistent [memory](/features/memory), use [tools and skills](/features/tools-and-skills) automatically when they help, and pull live context from [connected apps](/features/connectors). Auto mode can choose available capabilities when they seem useful.

**API usage billing.** A new monthly [API usage budget](/features/usage-budget) replaces fixed message limits. Track what you've used, see what counts, and know when it resets — all from the chat sidebar.

## Updates

**Clearer context indicator.** The [context window](/features/context-window) tracker now updates more reliably during long chats so you can tell when Babbily will compact older turns.

**Budget refresh.** Your usage view refreshes faster after each turn so recent activity is reflected quickly; final usage may continue to settle as billing data refreshes.

## Fixes

- Resolved a stall when the context-limit warning appeared mid-send.
- Restored model fallback so [chat](/features/chat-and-models) keeps working when a preferred model is briefly unavailable.
- Fixed scrolling and layout glitches in [temporary chats](/features/temporary-chat) and on tablet viewports.
- Sign-in via Google and Apple is now more reliable on mobile.

</Update>
